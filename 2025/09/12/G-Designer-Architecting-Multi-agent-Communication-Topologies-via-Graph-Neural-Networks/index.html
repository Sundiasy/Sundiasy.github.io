Introduction

Difficulty: it challenging to effortlessly identify the most efficient and complexity-adaptive multi-agent topology for a given task.

Satisfy the rule that

(1)有效性：通信结构必须有效地产生 Q 的合格解；

(2)复杂性自适应性：拓扑应该动态地适应任务的复杂性，最大限度地减少通信开销；

(3)对抗健壮性：拓扑应该在对手攻击下保持可靠。

The contributions:

  • 该协议从性能、适应性和健壮性三个维度对多智能体的拓扑设计进行了全面的规范(MACP)，突出了已有设计的不足。
  • 实用的解决方案。我们提出了 G-Designer，它是一种高效、自适应和健壮的 LLM 驱动的多代理通信图设计器。G-Designer 利用变化图自动编码器构建和处理多智能体网络，解码任务自适应的高性能智能体通信，并通过动态拓扑调整对基于智能体的敌意攻击具有较强的稳健性。（新的 Agent 架构 G-Designer）

Experiment

实验部分旨在验证 G-Designer 在多个维度上的有效性。

Setting

  • 数据集: 涵盖三大类任务：通用推理: MMLU 数学推理: GSM8K, MultiArith, SVAMP, AQUA 代码生成: HumanEval
  • 基线模型: 单智能体方法: CoT, ComplexCoT, Self-Consistency, PHP；多智能体方法: Chain, Star, Tree, Complete Graph, Random Graph, AutoGen, MetaGPT, LLM-Debate, DyLAN, GPTSwarm
  • 实现细节: 主要使用 gpt-4 和 gpt-3.5 作为智能体的基础模型。锚点拓扑被设定为简单的链式结构。

Result

  1. 高性能 (High-performing):
    如论文中的 Table 1 所示，G-Designer 在 6 个基准测试中的 5 个上都取得了最佳性能。例如，在 HumanEval（代码生成）上，其 pass@1 分数达到 89.90%，超过了专门为代码任务设计的 MetaGPT 和其他先进的多智能体框架。

  2. 任务自适应性 (Task-adaptive):
    Figure 6 直观地展示了 G-Designer 的自适应能力。对于 简单任务（如 HumanEval case a: 实现 strlen 函数），G-Designer 生成了一个极简的 “算法设计师 → 程序员” 的两节点拓扑，剪除了所有不必要的智能体。对于 复杂任务（如 HumanEval case c: 实现一个代数表达式计算器），它生成了一个包含项目经理、算法设计师、程序员、测试员和 Bug 修复员的五节点复杂网络。
    这证明了 G-Designer 能够根据任务难度调整拓扑，从而在保证质量的同时，大幅减少不必要的通信和 Token 消耗。

  3. 经济性 (Token-economical):
    Figure 4 对比了不同方法在性能和 Token 消耗上的表现。结果显示，虽然像 DyLAN 和 GPTSwarm 这样的复杂方法性能很高，但它们的 Token 消耗巨大。相比之下，G-Designer 在所有四个展示的数据集上都实现了 最高的性能和最低的 Token 消耗，达到了效率和效果的最佳平衡。

  4. 鲁棒性 (Adversarially robust):
    Figure 5 展示了在对其中一个智能体进行提示词攻击（prompt attack）前后的性能变化。许多简单拓扑（如 Chain, Complete Graph）和其他一些框架（如 AutoGen, DyLAN）在攻击后性能大幅下降（高达 11%）。而 G-Designer 的性能几乎不受影响，准确率仅下降了 0.3%，表现出与专门优化过的 GPTSwarm 相当的强大鲁棒性。

  5. 消融实验 (Ablation Study):
    Table 3 通过移除 G-Designer 的不同组件来验证其重要性。移除 虚拟任务节点 (w/o v_task) 导致性能下降最为显著，证明了任务感知能力是其成功的关键。移除 稀疏性正则化 (w/o SR) 会使系统在面对攻击时变得更脆弱。移除 锚点拓扑 (w/o Anchor) 也会导致性能下降，说明基于先验知识的引导是有益的。

Conclusion

最终的策略梯度思想很精彩，以后可以进行深入学习。
