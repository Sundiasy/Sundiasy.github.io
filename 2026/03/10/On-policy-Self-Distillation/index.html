On-Policy Self-Distillation for Reasoning Compression

1. 这篇论文试图解决什么问题？

这篇论文主要解决的是推理模型（Reasoning Models）过度冗长（Verbosity）的问题。

-

现状：现代推理模型（如 OpenAI o1、Gemini 2.5、DeepSeek-R1、Qwen3 等）在给出答案前，会生成数千个 token 的内部思考过程 。

-

痛点：虽然这种“长篇大论”在解决难题时很有效，但模型即使在面对如计算 2+2 这样极其简单的问题时，也停不下来，可能耗费 500 个 token 去思考 。

-

深层危害：大部分推理模型生成的内容不仅仅是冗余，而且是“有害的噪声” 。每一个不必要的 token 都有可能偏离正确方向，引发并放大多步推理中的级联错误（Compounding error） 。

2. 有哪些相关研究？

学术界已经注意到了这个问题，并提出了多种压缩推理过程的方法，但每种方法都存在明显的妥协：

-

强化学习（RL）+ 长度惩罚：在奖励函数中对输出长度进行惩罚 。缺点：必须依赖有标准答案（Ground-truth）的数据才能训练，并且会无差别地压制模型探索问题的能力（导致熵坍塌） 。

-

监督微调（SFT）：收集别人较短的推理轨迹，然后让模型去模仿 。缺点：会导致分布偏移（Distribution shift），学生模型在学习别人的推理方式时，会遗忘自己原本的推理能力 。

-

提示词/无训练压缩（Prompting/Pruning）：通过修改提示词让模型输出草稿或在推理后进行剪枝 。缺点：压缩效果有限，一旦拿掉特定的提示词，模型又会恢复冗长 。

-

以往的同策略自蒸馏（On-policy self-distillation）：将模型作为自己的老师 。缺点：此前的研究依然需要将标准答案作为特权信息提供给老师模型 。

3. 论文如何解决这个问题（核心方法）？

论文提出了一种名为 OPSDC (On-Policy Self-Distillation for Reasoning Compression) 的方法。它的核心理念极其简单：模型其实已经知道如何简明扼要地回答问题，它只是需要“被允许”这么做 。

具体解决步骤和数学原理如下：

第一步：构建老师与学生（同源同宗）

-

老师模型 (Teacher)：，这是附加了“简洁指令”（指令 例如：“请简洁作答，避免不必要的步骤”）的模型 。

-

学生模型 (Student)：，这是没有附加压缩指令的原始模型 。

  • 它们共享参数，但接收不同的上下文输入 。老师模型不需要标准答案 。

第二步：优化目标（反向 KL 散度） 训练过程是让学生模型生成回答，并在 Token 级别上最小化学生和老师分布之间的反向 KL 散度（Reverse KL divergence） 。公式如下：

$$\mathcal{L}(\theta) = \mathbb{E}{x\sim\mathcal{D}, y\sim\pi{\theta}(\cdot|x)} \left[ \sum_{t=1}^{|y|} D_{KL}(\pi_{\theta}(\cdot|x,y_{<t}) || \pi_{\bar{\theta}}(\cdot|x,c,y_{<t})) \right]$$

-

细节重点：为什么用反向 KL 而不是正向 KL？因为反向 KL 的梯度是由学生模型当前的概率分布加权的，这意味着学生只在它自己生成的区域进行微调，这提供了一种天然的自我正则化，防止了模型准确率的崩溃（Accuracy collapse） 。

第三步：周期性更新老师（Progressive Compression） 为了防止学生学完之后就停滞不前，论文采用了一种动态更新策略。每训练 步（例如 ），就把当前学生的参数同步给老师： 。

  • 因为学生已经学会了更简洁，更新后的老师加上简洁指令 会产出更加简洁的分布，从而推动学生不断自我压缩 。

理论解释（隐式奖励）：

这种做法在数学上等价于最大化一个“隐式奖励”：

当老师比学生更倾向于某个 Token 时，这个奖励就是正的，从而在不需要人工设计奖励函数的情况下，引导模型走向简洁和正确 。

4. 论文做了哪些实验？

-

实验设置：使用 Qwen3-8B 和 Qwen3-14B 模型 。在不包含标准答案的约 13,600 个数学竞赛题目（来自 DAPO-Math-17k）上进行单 epoch 训练 。

-

测试基准：MATH-500、AIME 2024、AIME 2025 。

  • 核心结果：

    -

    大幅压缩与准确率提升：在 MATH-500 上，模型减少了 57% 到 59% 的 Token 数量，同时准确率不仅没降，反而绝对提升了 9 到 16 个百分点（达到约 86%） 。在 AIME 2024 上，14B 模型压缩了 41%，准确率提升了 10.5 个百分点 。

    -

    自适应难度：方法自动展现出了对难度的适应性。面对简单的 MATH-500 压缩得很狠（约 57-59%），面对极难的 AIME 2025 压缩得相对保守（约 35%） 。

    -

    保持探索性（熵不变）：与传统的 RL 长度惩罚不同，OPSDC 训练后的模型熵值保持稳定，没有丧失复杂的试错思考能力 。

    -

    通用能力未受损：训练后的 MMLU 分数完全保持原样 。

5. 有什么可以进一步探索的点？

论文在结尾提出了几个很有价值的未来探索方向：

-

能力阈值研究（Capability threshold）：越聪明的模型（如 14B 比 8B）从这种蒸馏中获益越多，因为它们遵循指令的能力更强 。未来可以探究，基础模型需要达到怎样的“最低能力门槛”，才能有效进行自我蒸馏 。

-

跨领域应用：因为 OPSDC 不需要标准答案，只需要问题提示词和简洁指令，所以它非常适合扩展到代码生成（Code generation）或科学问答（Scientific QA）等难以获取高质量验证器或答案的领域 。

-

细粒度评估老师质量：进一步探索在不同模型家族中，“简洁指令”在什么时候、为什么会提升或降低准确率，以加深对自蒸馏动力学的理解 。

6. 总结一下论文的主要内容

这篇论文打破了以往“推理越长、结果越好”的迷思。研究表明，推理模型的冗长往往不是出于谨慎，而是引发累积错误的温床 。

通过提出 OPSDC 方法，作者证明了模型本身就具备简洁思考的潜能。利用“加上简洁指令的模型”作为老师，并通过计算反向 KL 散度去指导原始学生模型，OPSDC 实现了一个优雅的闭环：不需要任何标准答案、不需要硬性的长度预算，就能让模型自动学会对简单问题“快刀斩乱麻”，对复杂问题“深思熟虑” 。最终，这不仅让模型思考得更少，还让它们思考得更准。
