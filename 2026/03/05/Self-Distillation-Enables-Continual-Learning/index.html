Self-Distillation Enables Continual Learning

rrsun Lv3
  2026-03-05 15:28:12 2026-03-05 15:28:12 Created   2026-03-05 15:30:21 2026-03-05 15:30:21 Updated  
  • paper reading 
 
  • OPD 
 

这篇论文的核心逻辑非常优雅：让模型利用自身的“上下文学习（ICL）”能力，通过给昨天的自己当老师，来解决困扰大模型已久的“灾难性遗忘”问题。

1. 这篇论文试图解决什么问题？

目前，基础模型在部署后往往是“静态”的 。虽然可以通过检索或提示词改变行为，但无法在不更新参数的情况下内化新知识 。 然而，传统的微调（如 SFT）在引入新技能时，会面临以下核心挑战：

  • 灾难性遗忘 (Catastrophic Forgetting)： 当模型学习新任务时，其在旧任务或通用任务上的表现会剧烈下降 。

  • 离策略学习的局限： 监督微调（SFT）本质上是 离策略（Off-policy） 的，即模型在模仿外部专家数据，而不是在修正自己的错误 。

  • 奖励函数的缺失： 虽然 在策略（On-policy） 强化学习（RL）能显著减少遗忘，但它通常需要明确的奖励函数（Reward Function），这在现实中很难获得 。

这篇论文提出的 SDFT (Self-Distillation Fine-Tuning) 试图在只有专家演示（Demonstrations）的情况下，实现高效、低遗忘的持续学习 。

2. 有哪些相关研究？

作者梳理了三大类技术路径，并指出了它们的不足：

  • 在策略 vs 离策略学习： 研究表明，在策略更新（模型在自己生成的轨迹上学习）能显著降低遗忘 。但以往这种方法大多局限于需要奖励函数的 RL 领域 。

  • 逆强化学习 (IRL)： 理论上可以从演示中推断奖励再进行 RL 。但 IRL 极难扩展，通常需要极强的先验假设（如 RLHF 中对偏好的假设）。

  • 上下文蒸馏 (Context Distillation)： 这种方法让带有上下文信息的模型指导不带信息的模型 。但以往的方法通常是静态且离线的，无法根据每个具体的 Query 动态调整 。

3. 论文如何解决这个问题？（核心方法 SDFT）

SDFT 的天才之处在于：它将大模型的 ICL 能力视作一种“隐含的奖励函数” 。

3.1 核心机制：模型分身

对于每一个查询 ，模型同时扮演两个角色 ：

  1. 学生 (Student)： 仅根据查询生成分布 。

  2. 老师 (Teacher)： 将 相同的模型 与专家演示 拼接。通过 ICL，老师产生一个更优的、感知演示的分布 。

3.2 训练目标：在策略蒸馏

训练的目标是最小化学生和老师之间的 逆向 KL 散度 (Reverse KL Divergence) ：

$$\mathcal{L}(\theta)=D_{KL}(\pi_{\theta}(\cdot|x)||\pi(\cdot|x,c))=\mathbb{E}{y\sim\pi{\theta}(y|x)}\left[\log\frac{\pi_{\theta}(y|x)}{\pi(y|x,c)}\right]$$

为什么要在策略？ 因为 是从学生策略（当前模型）中采样出来的 。这意味着模型是在自己生成的答案上进行对比和修正。

3.3 数学证明：SDFT 逆强化学习 (IRL)

作者证明了，这个蒸馏过程等价于在最大化一个 隐式奖励 ：

简单来说，如果一个 Token 能让模型从“普通模式”切换到“看到演示后的高手模式”的可能性增加，它就会获得正奖励 。

3.4 关键技术细节

  • EMA 教师： 为了稳定训练，老师模型的权重采用学生参数的指数移动平均 (EMA)，而不是每一轮都实时更新 。

  • 解析梯度： 使用解析化的每 Token KL 梯度估计器，这比纯采样的方法更稳定 。

4. 论文做了哪些实验？

作者在 Qwen-2.5-7B/14B 和 Olmo-3-7B 等模型上进行了多维度验证 。

4.1 任务设置

  1. 技能学习 (Skill Learning)： 科学问答 (Science Q&A)、工具使用 (Tool Use)、医疗推理 (Medical) 。

  2. 知识获取 (Knowledge Acquisition)： 注入 2025 年发生的自然灾害等新知识（模型预训练 cutoff 之后的数据）。

4.2 实验结论

  • 帕累托最优： 相比 SFT 和传统的蒸馏方法，SDFT 在“提升新任务准确率”和“保留旧任务能力”之间取得了最佳平衡 。

    • 例如，在 Tool Use 任务中，SFT 会导致旧能力大幅下降，而 SDFT 几乎维持了基座水平 。

  • 克服“思维崩溃”： 在训练思维模型（Think Models）时，SFT 往往会导致模型生成的推理链变短、性能下降；而 SDFT 能保住模型原有的推理深度 。

  • 模型规模效应： 实验发现，模型越大（ICL 能力越强），SDFT 的优势就越明显。14B 模型的领先幅度远超 3B 模型 。

  • 串行积累能力： 在连续学习三个任务的极难设置下，SDFT 实现了能力的平滑积累，而 SFT 的表现呈现“锯齿状”，学了后面忘了前面 。

5. 有什么可以进一步探索的点？

  • 处理噪声演示： 目前 SDFT 依赖高质量的专家演示，如果演示数据有噪声，老师可能会教错 。

  • 解决语言偏差： 训练中发现学生可能会学到老师的一些口头禅（如“根据提供的例子…”），目前需要用 Mask 掉前几个 Token 的 Trick 来解决 。

  • 协同 RL： SDFT 可以作为 RL 的初始阶段，先通过自蒸馏打好基础，再引入外部奖励信号 。

6. 总结

SDFT 提供了一种全新的思路： 它证明了模型内部已经蕴含了学习新知识所需的“进化方向” 。通过将 ICL 与在策略蒸馏结合，它不需要昂贵的奖励工程，就能让大模型像人类一样，在不遗忘过去的前提下，不断吸收新技能、内化新知识 。
