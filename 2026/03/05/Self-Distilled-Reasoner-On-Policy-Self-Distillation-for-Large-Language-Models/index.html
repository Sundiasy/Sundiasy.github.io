Self-Distilled Reasoner: On-Policy Self-Distillation for Large Language Models

rrsun Lv3
  2026-03-05 15:26:09 2026-03-05 15:26:09 Created   2026-03-05 15:35:13 2026-03-05 15:35:13 Updated  
  • paper reading 
 
  • OPD 
 

这篇论文的核心逻辑非常迷人：它探讨了模型是否可以像人类学生一样，通过“看标准答案（特权信息）”来反思并教导那个“没看答案的自己”。

1. 这篇论文试图解决什么问题？

在提升大语言模型（LLM）推理能力的道路上，目前主要有三条路径，但各有痛点：

-

强化学习（RLVR，如 GRPO）： 虽然效果好，但极其耗费计算资源（采样成本高），且奖励信号过于稀疏（只有对错，没有过程指导） 。

-

监督微调（SFT）： 依赖高质量推理轨迹，容易产生“曝光偏差（Exposure Bias）”，即模型在推理时一旦走偏就无法自我纠正 。

-

传统知识蒸馏（Off-policy Distillation）： 通常需要一个更大、更强的外部教师模型，且由于训练数据分布与推理分布不一致，容易导致误差累积 。

本论文的目标： 提出一种**在线自我蒸馏（On-Policy Self-Distillation, OPSD）**框架，让模型在不需要外部教师、不依赖昂贵采样的情况下，实现高效率的推理能力进化 。

2. 有哪些相关研究？

论文立足于以下几个研究领域的交汇点：

-

在线蒸馏（On-Policy Distillation）： 如 GKD 等方法，让学生在自己生成的路径上接受教师指导，解决分布不匹配问题 。

-

验证奖励的强化学习（RLVR）： 如 DeepSeek 提出的 GRPO，通过组内相对奖励来优化模型，但存在采样效率低的问题 。

-

模型自我改进（Self-Training）： 如 STaR 和 ReST，通过生成推理过程、过滤正确答案并微调来迭代更新 。

-

特权信息学习： 借鉴了“评估比生成容易”的直觉，利用测试时不可见的信息来辅助训练 。

3. 论文如何解决这个问题？

论文提出了 OPSD（On-Policy Self-Distillation） 框架。其核心思想是：同一个模型，两种身份 。

3.1 核心机制：身份切换

模型 根据输入上下文的不同，扮演两个角色：

1.

学生策略 ()： 只看到问题 。这是推理时真实的状态 。

2.

教师策略 ()： 看到问题 和特权信息 （如标准答案或参考推理链） 。

$$p_T(\cdot | x, y^) := p_{\theta}(\cdot | x, y^)$$

3.2 训练流程

1.

在线采样： 学生策略针对问题 采样生成一条推理轨迹 。

2.

双重评估： 对于生成的每一个 token，教师和学生都给出其预测概率分布。教师因为由于有“标准答案”护航，能给出更精准的指导 。

  1. 损失函数： 计算教师和学生在学生生成的轨迹上的逐 Token 散度。

3.3 数学表达

训练的目标是最小化以下损失函数 ：

$$\mathcal{L}{OPSD}(\theta) = \mathbb{E}{(x, y^) \sim \mathcal{S}} \left[ \mathbb{E}{\hat{y} \sim p_S(\cdot | x)} \left[ \frac{1}{|\hat{y}|} \sum{n=1}^{|\hat{y}|} D(p_T(\cdot | x, y^, \hat{y}{<n}) || p_S(\cdot | x, \hat{y}{<n})) \right] \right]$$

其中， 通常采用 Jensen-Shannon 散度 () 。

注意： 梯度只流向学生端的 logits，教师端在计算时视为固定目标（Fixed Target） 。

3.4 为什么这比 GRPO 强？

-

稠密信号： GRPO 只在序列末尾给一个 0 或 1 的奖励；OPSD 在每个 token 处都有教师给出的全概率分布指导 。

-

极高效率： OPSD 只需要 1 个 rollout（采样），而 GRPO 通常需要 8 个或更多来估计优势函数 。

4. 论文做了哪些实验？

作者在 Qwen3 系列模型（1.7B, 4B, 8B）上进行了数学推理实验 。

4.1 性能表现

在 AIME24、HMMT25 等高难度数学竞赛榜单上，OPSD 显著优于 SFT，并在 4B/8B 规模上达到或超过了 GRPO 的水平 。

4.2 核心发现

-

Token 效率（降本增效）： OPSD 展现了 4-8 倍于 GRPO 的 token 效率 。这意味着达到同样的精度，OPSD 跑得更快、更省钱。

-

模型规模效应： 在 1.7B 模型上效果提升有限，但在 4B 和 8B 上提升显著 。这证实了我们的假设：模型必须达到一定的“智力门槛”，才能有效地“自我合理化”标准答案并教导自己 。

-

全词表蒸馏 vs. 采样蒸馏： 实验证明，匹配整个词表的概率分布（Full-vocabulary logit distillation）比只看采样到的 token 效果更好 。

5. 有什么可以进一步探索的点？

这篇论文虽然强大，但也留下了不少脑洞：

-

更大规模的验证： 目前实验只做到 8B，如果放到 70B 甚至更大的模型上，这种“自我反思”的能力是否会产生质变？

-

引入正确性信号： 目前 OPSD 只做分布匹配，并没有显式地告诉模型“这步错了”。如果结合判别器（Outcome Reward）可能会更强 。

-

课程学习： 并非所有题目都适合自学。如果题目太难，教师也教不会自己。引入“由易到难”的自动题目筛选机制（Curriculum Learning）非常有前景 。

6. 总结

OPSD 论文的核心贡献在于： 它打破了“蒸馏必须有强教师”和“强化学习采样太贵”的僵局。它证明了：只要给 LLM 一个看答案的机会，它就能通过内部的“特权上下文”产生高质量的监督信号，实现极具性价比的自我进化 。

论文金句： “Rationalization is easier than generation.” (解释答案比想出答案容易) —— 这是 OPSD 成立的哲学基石 。
